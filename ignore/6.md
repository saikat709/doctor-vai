## 2. Technical Component Specifications

### A. Triage Form Structure

Build the entry interface using high-contrast input components. Force strict entry types across the following data attributes:

* **Patient Age:** Numerical input (`type="number"`).
* **Gender Selector:** Explicit select elements (`"Male" | "Female" | "Other"`).
* **Severity Tag:** Low-resource visual priority dropdown (`"Low" | "Medium" | "High"`).
* **Comorbidities:** Text area descriptor handling multi-disease history arrays.
* **Symptoms Intake:** Comprehensive, multiline text zone capturing full presenting physical logs.

### B. DeepSeek Guardrail Orchestration Engine

* **API Endpoint Delivery:** Create or route payloads directly through `/api/diagnostics/risk`.
* **Strict Prompt Boundary Enforcements:** The system prompt sent to DeepSeek must be strictly bounded to prevent medical hallucinations. It must command the LLM:
1. *Never assign a definitive name-based disease diagnosis.* (e.g., Do not say: "The patient has peripheral artery disease" or "The patient has cancer").
2. Evaluate the aggregated data points purely to categorize the immediate care response footprint into one of three strict status metrics: `Standard Care`, `Monitor Closely`, or `High Priority`.
3. Output actionable clinical metrics and preventative safety precautions structured strictly into clean arrays.



### C. Visual Triage Status Mapping

The UI layout must capture the output status and apply high-contrast banner highlights matching these conditions:

| Attention Flag State | UI Color Token Accent | Operational Field Status Definition |
| --- | --- | --- |
| `Standard Care` | `bg-blue-600` | Normal field protocol; standard baseline tracking. |
| `Monitor Closely` | `bg-amber-500` | Elevate assessment frequency; track specific vitals. |
| `High Priority` | `bg-rose-600 animate-pulse` | High-frequency alert state; prepare immediate escalation routing. |
