## Overview

When `OFFLINE=true` in `.env`, the app runs as an Electron desktop application with:
- Embedded **PGlite** (pure WASM PostgreSQL, zero native deps, works offline) as the local database
- No external DB connection (DATABASE_URL ignored)
- Ollama (already configured via Phase 2 settings) as the only LLM backend
- Full Prisma migration support against PGlite
- Settings page enforces Ollama config on first launch if not already set

## 2. Environment Setup

In `.env`:
```
OFFLINE=true   # set to false or omit for online/cloud mode
```

Create `lib/env.ts`:
```ts
export const isOffline = process.env.OFFLINE === "true"
```

This flag gates:
- Which DB client is used (PGlite vs Neon/Postgres)
- Whether Ollama is required on startup
- Whether cloud auth is bypassed (offline uses a local session)

```ts
// If isOffline:
//   Use PGlite via a Prisma custom adapter (see step 4)
```

Update ALL existing files that import `PrismaClient` directly to import `db` from `lib/db.ts` instead.


```ts
import { PGlite } from "@electric-sql/pglite"
import { PrismaPg } from "@prisma/adapter-pg"
import { PrismaClient } from "@prisma/client"

const pglite = new PGlite(dbPath) // dbPath resolved per context
const adapter = new PrismaPg(pglite)
export const db = new PrismaClient({ adapter })
```

## 6. Electron Setup

### `electron/main.ts`

```ts
import { app, BrowserWindow, dialog } from "electron"
import { join } from "path"
import { migrateOffline } from "../lib/migrate-offline"

async function createWindow() {
  // 1. If OFFLINE=true, run migrations first
  if (process.env.OFFLINE === "true") {
    try {
      await migrateOffline()
    } catch (e) {
      dialog.showErrorBox("Startup Error", "Database migration failed. Please restart.")
      app.quit()
      return
    }
  }

  // 2. Create BrowserWindow
  const win = new BrowserWindow({
    width: 1280,
    height: 800,
    minWidth: 900,
    minHeight: 600,
    title: "Health Worker Assistant",
    icon: join(__dirname, "../public/icon.png"),
    webPreferences: {
      preload: join(__dirname, "preload.js"),
      contextIsolation: true,
      nodeIntegration: false,
    },
  })

  // 3. Load Next.js
  // In dev: wait for Next.js dev server then loadURL("http://localhost:3000")
  // In prod: use next-electron-server or serve the exported static build
  const isDev = !app.isPackaged
  if (isDev) {
    win.loadURL("http://localhost:3000")
    win.webContents.openDevTools()
  } else {
    win.loadFile(join(__dirname, "../out/index.html"))
  }
}

app.whenReady().then(createWindow)
app.on("window-all-closed", () => { if (process.platform !== "darwin") app.quit() })
```

### `electron/preload.ts`

```ts
// Minimal preload — expose nothing sensitive
// Only expose: window.electronAPI = { platform: process.platform }
import { contextBridge } from "electron"
contextBridge.exposeInMainWorld("electronAPI", {
  platform: process.platform,
})
```

## 8. Offline Auth — Bypass Cloud Auth

When `OFFLINE=true`, Auth.js cloud OAuth/credentials are irrelevant. Instead:

Create `lib/offline-session.ts`:
```ts
// Returns a hardcoded local session:
// { user: { id: "local-user", name: "Field Worker", email: "local@offline" }, expires: "2099-01-01" }
// This is used wherever getServerSession() / auth() is called
```

Update `lib/auth-helper.ts` (create if not existing):
```ts
import { isOffline } from "./env"
import { getOfflineSession } from "./offline-session"
import { auth } from "@/auth"

export async function getSession() {
  if (isOffline) return getOfflineSession()
  return await auth()
}
```

Replace all `auth()` / `getServerSession()` calls in API routes with `getSession()` from this helper.
