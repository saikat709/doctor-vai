## TASK: Connect Local LLM via Ollama + Cloudflare Tunnel

### Overview

In the Settings page, add a **"Connect Local"** button with a Shield/Lock icon. Clicking it opens a multi-step modal that guides the user through:
1. Selecting their OS platform
2. Installing Cloudflared
3. Running Ollama with a model
4. Setting up a Cloudflare tunnel to local Ollama (port 11434)
5. Entering the tunnel URL → verifying connectivity → fetching available models
6. Selecting a model to use

Once configured, ALL AI/LLM calls in the app use the local Ollama model instead of DeepSeek, based on a flag stored in the DB.

---

## 1. Prisma Schema Update

Add a `LocalLLMConfig` model and update `User`:

```prisma
model LocalLLMConfig {
  id            String   @id @default(cuid())
  userId        String   @unique
  user          User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  enabled       Boolean  @default(false)
  tunnelUrl     String   // e.g. https://abc123.trycloudflare.com
  selectedModel String   // e.g. llama3, mistral, phi3
  verifiedAt    DateTime?
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}
```

Add relation to `User`:
```prisma
localLLMConfig  LocalLLMConfig?
```

Run migration:
```bash
npx prisma migrate dev --name add_local_llm_config
```

## 4. API Routes

### `POST /api/settings/local-llm/verify`

```ts
// Body: { tunnelUrl: string }
// Auth-guarded
// 1. Sanitize and validate the URL
// 2. Call `${tunnelUrl}/api/tags` (Ollama's model list endpoint)
//    - Use fetch with a 8 second timeout
// 3. If successful: return { success: true, models: string[] }
//    - Parse response: models = data.models.map(m => m.name)
// 4. If fetch fails or times out: return { success: false, error: "Tunnel not reachable" }
// Never expose internal errors to client
```

### `POST /api/settings/local-llm/save`

```ts
// Body: { tunnelUrl: string, selectedModel: string }
// Auth-guarded
// Upsert LocalLLMConfig for the current user:
//   { enabled: true, tunnelUrl, selectedModel, verifiedAt: new Date() }
// Return: { success: true }
```

### `DELETE /api/settings/local-llm/save`

```ts
// Auth-guarded
// Set LocalLLMConfig.enabled = false for current user
// Return: { success: true }
```

## 5. LLM Client Abstraction

Create `lib/llm-client.ts`:

```ts
// getLLMClient(userId: string): Promise<OllamaClient | DeepSeekClient>
//
// 1. Fetch user's LocalLLMConfig from DB
// 2. If config exists, enabled === true, tunnelUrl and selectedModel are set:
//    → return an Ollama client configured with baseURL = tunnelUrl
//    → use the `ollama` npm package: import Ollama from 'ollama'
//      const ollama = new Ollama({ host: tunnelUrl })
// 3. Otherwise:
//    → return the existing DeepSeek/OpenAI-compatible client
//
// Export a unified `generateCompletion(userId, messages, systemPrompt)` function
// that internally uses whichever client is active and returns { content: string }
```

Install the Ollama JS SDK:
```bash
npm install ollama
```

The unified function signature:
```ts
export async function generateCompletion(
  userId: string,
  messages: { role: "user" | "assistant"; content: string }[],
  systemPrompt: string
): Promise<{ content: string }>
```

For Ollama calls use `ollama.chat()`. For DeepSeek use the existing OpenAI-compatible SDK call.
