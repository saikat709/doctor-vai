# Treatment Guidance Engine, Dynamic JSON Parsing & Procedural Checklist View

This document provides deep technical layout rules and specific code architecture guidelines to implement the interactive Treatment Guidance page (`app/(dashboard)/guidance/page.tsx`). The primary design goal is absolute clarity: field workers under intense pressure must be able to parse complex clinical workflows in seconds, tracking completed operational steps without losing their place.

## 2. Technical Component Specifications

### A. Data Layer Integration (`/lib/procedures.json`)

The system must parse a localized JSON dictionary. The file must follow a predictable, strictly typed structural schema matching this format:

```json
{
  "procedures": [
    {
      "id": "neonatal-asphyxia",
      "title": "Neonatal Asphyxia Management",
      "category": "Neonatal",
      "severity": "High",
      "steps": [
        "Clear airway using mechanical bulb suction tool.",
        "Initiate positive pressure ventilation (PPV) within the first golden minute.",
        "Monitor heart rate and chest rise continuously every 30 seconds."
      ]
    }
  ]
}

```

### B. State Management & Animated Checklist Tracker

* **Active Selection Memory:** Use React client state hooks (`useState`) to toggle between the grid layout overview and the deep-dive procedure checklist view without requiring secondary page reloads.
* **Progress Tracking Caching:** Implement checkbox components next to each step. When a user clicks a step, toggle its completion state and cache it in the active component state so workers have a visual paper trail as they work through an emergency.
* **Micro-Animations (Framer Motion):**
* **Staggered Entrances:** When a procedure opens, animate the list items sequentially (`staggerChildren` delay at 0.05s) rather than slamming them all on the screen at once. This guides the worker's eye naturally from Step 1 down to the end.
* **Visual Striking:** When a checkbox is checked, use motion layouts to cleanly fade and strike through the text, making it clear what is finished and what is pending.
