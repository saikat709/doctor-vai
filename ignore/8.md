
# Immunization Tracking Ledger, Urgency-Sorted Scheduling Matrix & Visual Reminders

This document provides production-grade engineering rules and technical specifications to construct the Vaccination Reminder tracking system (`app/(dashboard)/reminders/page.tsx`). The primary goal is to build an uncompromised, high-visibility ledger that automatically categorizes patient immunization records by urgency (Overdue, Pending, Completed), eliminating the risk of missed doses in remote clinics.

## 2. Technical Component Specifications

### A. Core Schedule Architecture (`app/(dashboard)/reminders/page.tsx`)

* **The Log Form:** Implement form components using clean date picker components for both the `administeredDate` and `nextDoseDueDate`. The fields map directly to your PostgreSQL database properties managed via Prisma.
* **Urgency Sorting Engine:** The UI table must not display items randomly or by creation order. It must apply a rigorous runtime logical sort:
1. `Overdue` items are locked to the top tier, sorted by the highest number of overdue days.
2. `Pending` items follow immediately below, sorted chronologically with the nearest upcoming due dates prioritized.
3. `Completed` items are rendered at the bottom or hidden behind an filter view.



### B. Dynamic Time-Delta Calculations

Rather than forcing field workers to calculate calendar dates manually, the UI component must compute the difference between the current date and the `nextDoseDueDate` at runtime:

* If `currentDate` is greater than `nextDoseDueDate`, calculate the delay magnitude: `X Days Overdue`.
* If `currentDate` is less than `nextDoseDueDate`, calculate the remaining window: `Y Days Remaining`.

### C. Visual Alert Level Tiers

The application row or badge element must read the computed record state and apply clear high-contrast markers:

| Computed Record State | Tailwind Color Token | Actionable Definition for Field Workers |
| --- | --- | --- |
| `OVERDUE` | `bg-rose-100 text-rose-700 animate-pulse` | Critical target date missed. Requires immediate patient outreach. |
| `PENDING` | `bg-amber-100 text-amber-700` | Scheduled upcoming dose window; stable tracking state. |
| `COMPLETED` | `bg-emerald-100 text-emerald-700` | Full dose compliance reached for this operational cycle. |
