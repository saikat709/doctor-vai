## 2. Auth.js v5 (NextAuth) + Prisma Integration

### Step A: Update Environment Variables (`.env.local`)

Generate a secure authentication token secret using `openssl rand -base64 32` and append the parameters:

```env
DATABASE_URL="postgresql://user:password@localhost:5432/health_worker_db?schema=public"
AUTH_SECRET="your_generated_base64_secret_key"

# DeepSeek API Configuration
DEEPSEEK_API_KEY="your_deepseek_api_credential_key"

```

### Step B: Database Client Singleton (`lib/db.ts`)

Avoid running into active instance leaks across developer continuous reloads:

```typescript
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as { prisma: PrismaClient | undefined };

export const db = globalForPrisma.prisma ?? new PrismaClient();

if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = db;

```

If this fails, try using neondb sdk, because we have neondb url.


### Step C: Core Auth Config Shell (`auth.config.ts`)

Isolate edge-safe configurations (like callbacks and route redirection logic) from the primary database package:

```typescript
import type { NextAuthConfig } from "next-auth";
import Credentials from "next-auth/providers/credentials";

export const authConfig = {
  pages: {
    signIn: "/login",
  },
  callbacks: {
    authorized({ auth, request: { nextUrl } }) {
      const isLoggedIn = !!auth?.user;
      const isOnDashboard = nextUrl.pathname.startsWith("/dashboard");
      
      if (isOnDashboard) {
        if (isLoggedIn) return true;
        return false; // Direct unauthenticated actors straight back to /login
      }
      return true;
    },
    jwt({ token, user }) {
      if (user) {
        token.id = user.id;
        token.role = user.role;
      }
      return token;
    },
    session({ session, token }) {
      if (session.user) {
        session.user.id = token.id as string;
        session.user.role = token.role as string;
      }
      return session;
    },
  },
  providers: [
    Credentials({
      async authorize(credentials) {
        // Implement simple fast field verification for clinical staff mapping
        // Mocking authentication check for amateur setup execution:
        if (credentials.email === "worker@health.gov" && credentials.password === "password123") {
          return { id: "1", name: "Field Medic Saikat", email: "worker@health.gov", role: "HealthWorker" };
        }
        return null;
      },
    }),
  ],
} satisfies NextAuthConfig;

```

### Step D: Primary Handler Hub (`auth.ts`)

Bind your configuration architecture safely together using the modern Prisma Adapter syntax:

```typescript
import NextAuth from "next-auth";
import { PrismaAdapter } from "@auth/prisma-adapter";
import { db } from "@/lib/db";
import { authConfig } from "./auth.config";

export const { handlers, auth, signIn, signOut } = NextAuth({
  adapter: PrismaAdapter(db),
  session: { strategy: "jwt" }, // Optimized strategy to ensure zero edge runtime crashes
  ...authConfig,
});

```
