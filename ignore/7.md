# Symptom-Based Disease Exploration, Tag Selector Matrix & Differential Diagnosis Pipeline

This document establishes the structural rules and technical guidelines to implement the Disease Detection module (`app/(dashboard)/diagnostics/disease/page.tsx` or integrated cleanly within your diagnostic hub view). The goal is to build an intuitive multi-tag selector matrix that communicates potential differential conditions clearly and ranks them by likelihood without presenting any single outcome as an absolute certainty.

## 2. Technical Component Specifications

### A. Symptom Tag Selector Matrix

* **The Tag Selector Grid:** To maximize operational speed on touch screens in low-resource environments, use an interactive multi-select checkbox or toggle-tag grid layout representing common local clinical presentations (e.g., *Fever, Cough, Chills, Fatigue, Diarrhea, Dyspnea, Jaundice, Abdominal Pain*).
* **State Aggregation:** Manage selected indicators within a local string array state (`string[]`). Provide a secondary text area input for custom notes, allowing field workers to add specific contextual details (e.g., duration, onset, severity patterns).

### B. Structured JSON Differential API (`app/api/diagnostics/disease/route.ts`)

* **Payload Packaging:** The POST call passes the aggregated `selectedSymptoms` array and the text description field directly to DeepSeek.
* **Strict JSON Extraction Requirements:** The system prompt must enforce a structured JSON schema mapping perfectly to your Prisma database storage configuration. Instruct the model to return a root `diagnoses` array containing exactly these keys:
```typescript
{
  diagnoses: Array<{
    diseaseName: string;
    likelihood: "High" | "Medium" | "Low";
    nextDiagnosticTest: string;
  }>
}

```



### C. Likelihood Component Architecture

The results table must read the categorical `likelihood` tag and wrap it inside a stark, color-coded badge component for instant visual assessment:

* `High`: High-contrast crimson/rose text badge (`bg-rose-100 text-rose-700 dark:bg-rose-900/30 dark:text-rose-400`).
* `Medium`: Alert amber/orange text badge (`bg-amber-100 text-amber-700 dark:bg-amber-900/30 dark:text-amber-400`).
* `Low`: Muted slate/blue text badge (`bg-slate-100 text-slate-700 dark:bg-slate-800 dark:text-slate-400`).
