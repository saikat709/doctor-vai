# Public Landing Page, Operational Hero Layout & Authentication Mock Router

This document details the production-grade implementation rules and technical blueprint to construct the application's root entry portal (`app/page.tsx`). The layout targets field workers operating under variable network bandwidths, demanding maximum loading speeds, stark scannability, and frictionless navigation transitions.

### Key Message Pillars to Include:

1. **Offline-First Resiliency:** Highlight that forms, session states, and clinical intake configurations remain active even in deep remote areas without stable cell service.
2. **Absolute Patient Data Privacy:** Explicitly state that clinical profiles and diagnostic queries remain localized, ensuring alignment with privacy protocols before syncing with central nodes.
3. **Local AI Decision Support:** Frame the assistant clearly as an advisory verification engine powered by an isolated vector knowledge base, not a primary diagnostic authority.

---

## 2. Technical Component Specifications

### A. Core Page Frame (`app/page.tsx`)

* **Authentication State Guarding:** Use the NextAuth `useSession()` client hook or an asynchronous `getServerSession()` server-side node logic blocks to check current authorization profiles.
* **Authenticated Path:** Immediately intercept the mount cycle and redirect the client to `/dashboard/session` using Next.js `redirect()` or `router.replace()`.
* **Unauthenticated Path:** Mount the landing grid layout with access buttons leading directly to the authentication portal.


* **The Master Call to Action (CTA):** A highly visible, prioritized main button ("Go to Dashboard"). It must feature a strong color accent (`bg-emerald-600 hover:bg-emerald-700` or `bg-sky-600`) with large tap targets (minimum `h-14` sizing layout metrics) to ensure seamless use on mobile touch interfaces in the field.

### B. Responsive Grid Features Matrix

* **Layout Configuration:** Implement a structural layout that shifts from a single stacked column on mobile viewports up to a clean 3-column layout on desktop screens (`grid grid-cols-1 md:grid-cols-3 gap-6`).
* **Visual Assets Profile:** Use clean borders, crisp typographic weight hierarchies, and minimal, lightweight vector illustrations (Lucide React icons) rather than heavy imagery. This guarantees the page loads within milliseconds over low-bandwidth field connections.
