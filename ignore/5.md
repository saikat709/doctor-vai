## 2. Technical Component Specifications

### A. Dynamic Input State Machine

* **The Array Builder:** The list of prescribed medications cannot be a simple static string field. Implement a dynamic React state array (`string[]`).
* **Interaction Controls:** Provide inline `[Remove]` buttons for each indexed item row, alongside an explicit `[+ Add Medicine]` button that pushes an empty string to the array, ensuring field workers can match the exact size of a patient's prescription stack.

### B. Structural Response Risk Stratification Matrix

When the form is submitted, the API payload returns an authoritative response that maps strictly to one of four visual alert levels. The UI configuration must match this exact design logic:

| API Returned Verdict | Tailwind Color Token | Visual Border & Backdrop Treatment |
| --- | --- | --- |
| `Go Ahead` | `bg-emerald-500` | Soft green tint tracking clean biochemical safety. |
| `Not Harmful` | `bg-sky-500` | Baseline informational sky tint; zero noted conflicts. |
| `Moderate Harm` | `bg-amber-500` | Alert orange. Displays clear precaution guidelines. |
| `Seems Harmful` / `Highly Dangerous!` | `bg-destructive` | High-frequency flashing red. Demands immediate stoppage. |

### C. DeepSeek Integration Framework (`app/api/interactions/route.ts`)

* **Payload Structuring:** The API route intercepts the POST transaction, reads the environment variables (`DEEPSEEK_API_KEY`), and issues a strict JSON-mode structure request to DeepSeek.
* **System Prompt Boundary Guardrails:** The prompt must instruct the model to return raw, clean JSON containing *exclusively* the following attributes without markdown wrapping:

```typescript
    {
      verdict: "Go Ahead" | "Not Harmful" | "Moderate Harm" | "Seems Harmful";
      explanation: string;
      alternatives: string[];
    }
    ```
